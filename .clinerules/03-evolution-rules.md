# 03 - 自进化触发规则

> 定义 MYcline 系统自我完善的触发条件和执行规则。
> 由 Cline 在每次任务结束后自动检查。

## 触发条件

### 条件 A：记忆冲突率过高
- **触发值**: `conflict_rate > 5%`
- **动作**: 重建记忆索引 → 去重合并冲突条目 → 记录到 evolution_log
- **来源**: memory_state.db 统计数据

### 条件 B：技能复用率过低
- **触发值**: `skill_reuse_rate < 20%`
- **动作**: 检查 skills/ 注册的脚本是否过时 → 更新或淘汰低效技能 → 生成新技能模板
- **来源**: memory_state.db 统计数据

### 条件 C：长时间无新技能沉淀
- **触发值**: 连续 3 天无新脚本写入 skills/
- **动作**: 回顾最近 3 天对话 → 从中挖掘可复用的方法/脚本模式 → 生成 skill 模板
- **来源**: 文件系统时间戳对比

### 条件 D：上下文命中率低
- **触发值**: `context_hit_rate < 60%`
- **动作**: 检查分类标签是否合理 → 优化 category_index.json → 重新标记历史任务
- **来源**: memory_state.db 统计数据

### 条件 D1：API集成测试（新增）
- **触发值**: 用户提供新API密钥或出现新MCP服务器可用
- **动作**: 执行API连通性测试 → 验证关键端点可用 → 记录到技能库
- **来源**: 用户提供或环境检测
- **示例**: 硅基流动API测试（DeepSeek-V3/V4, Qwen3, GLM-4等）

### 条件 E：定期健康检查
- **触发值**: 距离上次进化 >= 7 天
- **动作**: 执行完整记忆压缩 → 归档过期任务 → 生成系统健康报告
- **来源**: memory_state.db.last_evolution

### 条件 F：主动触发
- **触发值**: 用户输入 `/evolve-check` 或「运行自检」
- **动作**: 立即执行以上所有检查 → 输出综合报告

### 条件 G：上下文膨胀触发自优化
- **触发值**: Context Window Usage ≥ 60%
- **动作**:
  1. 自动生成**对话摘要**（记录已完成的步骤、关键决策、待办事项）
  2. 检查 `.clinerules/` 中所有规则 → 根据本次对话经验优化规则内容
  3. 检查 `prompts/` 对应分类的 prompt → 根据本次经验补充缺失的指令模式
  4. 压缩上下文 → 输出 `context_summary.txt` 到当前任务目录
  5. 用 `attempt_completion` 结项，重置上下文
- **来源**: 每次工具调用后的 Context Window Usage 自动检测

### 条件 H：任务完成强制总结
- **触发值**: 每次 `attempt_completion` 之前
- **动作**: 自动生成**本次任务摘要** → 写入当前任务目录的 `README.md`
- **摘要格式**:
  ```
  ## 任务摘要
  - 目标：[一句话]
  - 完成步骤：[列表]
  - 关键决策：[如有]
  - 产出文件：[路径列表]
  - 待办：[如有未完成项]
  - 经验教训：[本次学到可复用的模式]
  ```
## 执行流程

```
检测到触发条件
       │
       ▼
┌─────────────────┐
│ 读取 self_evolve │
│ .md 对应规则    │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 执行优化动作    │
│（如重建索引/    │
│  生成技能/      │
│  压缩日志等）   │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 验证优化结果    │
│  → 通过/回滚   │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 更新状态指标    │
│  → evolution_   │
│    count += 1   │
│  → last_        │
│    evolution =  │
│    现在        │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 生成进化报告    │
│ → evolution_log/│
│   YYYY-MM-DD.md │
└─────────────────┘
```

## 回滚机制

如果进化导致系统异常（如记忆检索变慢、分类错误增加）：
1. 自动检测异常指标
2. 回滚到上一次稳定状态
3. 记录失败原因到 evolution_log
4. 通知用户并提供手动修复选项
