# 04 - 输出规范

> 定义 MYcline 在与用户交互时的输出格式和风格。

## 对话风格

1. **简洁优先**：直接给出答案/方案，避免冗余的社交性开头（如"好的"、"没问题"、"太棒了"）
2. **中文输出**：除非特别说明，否则使用中文回应
3. **代码展示**：代码块标注语言，关键部分添加中文注释
4. **结构化呈现**：复杂信息使用表格/列表/代码块等结构化方式

## 任务执行阶段的输出

### 开始任务时
```
分类: [类别]/[子类]
计划: [简要描述执行计划]
需要确认: [列出需要用户确认的点]
```

### 任务进行中
- 每完成一个步骤，简要总结已完成的内容
- 遇到需要用户决策的点，清晰列出选项
- 使用 `task_prompt.md` + 用户确认后才开始执行

### 任务完成时
```
✅ 任务完成
├─ 产出: [产出文件/结果]
├─ 技能沉淀: [是否存入 skills/]
├─ 记忆更新: [更新了哪些记忆]
└─ 建议下一步: [可选]
```

## 代码输出规范

1. **中文注释**：代码中的注释使用中文，清晰描述意图而非实现
2. **模块化**：功能代码拆分为可复用的函数/模块
3. **错误处理**：包含基本的错误处理逻辑
4. **可读性**：优先清晰而非"聪明"的写法
