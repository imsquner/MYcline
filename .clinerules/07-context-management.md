# 07 - 上下文窗口管理策略

> 防止上下文窗口膨胀导致响应变慢或中断。
> Cline 每完成一步主动汇报，用户无需猜测"是否卡住"。

## 主动汇报规则（核心）

**每条消息末尾必须包含以下之一：**
- ✅ 刚完成了什么
- ⏳ 下一步要做什么
- ❓ 需要你决策什么

**这是防止你"以为我卡了"的唯一方法。** 我做完一步不说话 = 你会以为我死了。

## 上下文压缩策略

当 Context Window Usage > 60% 时，执行以下策略：

### 1. 优先用 `attempt_completion` 结项
- 阶段性任务完成后，立即用 attempt_completion 汇总结果
- 这样系统可以在新任务中重置上下文

### 2. 读完即忘，不保留大文件原文
- 读文件前记录路径，读完即丢原文
- 只保留"路径+做了什么修改"的摘要，方便调用时回溯路径

### 3. 拆分为独立子任务
- 大任务拆分为多个独立小任务
- 每个子任务完成后 attempt_completion
- 用户确认后再开始下一个

### 4. 引用替代粘贴
- 修改文件时只说"修改了文件X的第Y部分"
- 不粘贴完整文件内容
- 除非用户明确要求看

## 执行流程

```
用户发出指令
  ↓
我产出一个prompt用户确认
到
我执行一个工具
  ↓
立即汇报：✅ 第N步完成 / 当前状态
  ↓
用户看到消息 → 确认或继续
（不会出现"长时间没看到动静"的情况）
```

## 禁止行为

- ❌ 完成操作后沉默等待
- ❌ 一次读入多个大文件原文到上下文
- ❌ 一条消息内执行多个工具
