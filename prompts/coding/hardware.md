## 第一阶段：资料转换（资料→MD 格式）

**执行此步时，先向用户发出分类确认消息**

在执行任何设计之前，先将所有原始资料转换为可检索的 Markdown 格式：

```

方式C（图片型PDF）：如果提取为空（扫描件）
  → 记录需要 OCR，标记文件名为图片型
  → 尝试使用 skills/ppt_pdf_extractor.py
```

```
当前阶段：PDF中的原理图无法自动识别
处理方案：

  4. 标记「此处的原理图来源于XXX手册第Y页，文字已提取」

## 第四阶段：汇总设计文档（分两部分）

### 4.1 第一部分：BOM + 选型总表

按以下层级排列：
```
1. IC 类（主控、ADC、运放、基准、LDO）
2. 电阻类（精密电阻→普通电阻，按阻值排序）
3. 电容类（电解→陶瓷，按容值排序）
4. 电感/磁珠类
5. 连接器类（端子、排针、排母）
6. 其他（LED、开关、跳线、测试点）
```

每行必须包含：序号、位号、型号/规格、数量、封装、精度、作用

**列出所有元件的选型依据**：为什么选这个值、耐压、精度、封装。

### 4.2 第二部分：连接顺序说明（简洁版）

按信号流向逐级描述：
```
电源输入级：
  J1(24V输入) → 保险丝 → TVS管 → 输入电容(C1 10µF+C2 0.1µF)
  → 防反接二极管(D1 SS34) → U1(GD30DC1354 Pin3 VIN)
  
DC-DC转换级：
  U1(Pin8 BST) → C3(100nF 自举) → L1(10µH) → U1(Pin1 SW)
  U1(Pin4 FB) → R1(49.9kΩ 上) → R2(10kΩ 下) → GND
  
如此类推...
```

**原则**：
- 每一个连接节点用 `→` 箭头表达
- 关键参数写在括号里
- 器件位号大写加粗
- 每段不超过 5 行，太长就拆成子段

```

### 5.2 PCB 布线详细规则

确认通过后，按以下顺序和要求执行：

#### 5.2.1 布局顺序
```
1. 先放核心芯片（ADC/MCU/DC-DC）—— 中心位置
2. 再放去耦电容 —— 紧贴芯片电源引脚
3. 放置反馈/补偿网络 —— 远离噪声源
4. 放置输入/输出连接器 —— 板边
5. 放置滤波/保护元件 —— 靠近连接器
6. 放置辅助元件（LED、跳线）—— 剩余空间
```

#### 5.2.2 布线优先级
```
优先级1（必须）：电源走线 → 最短最粗，>40mil
优先级2（必须）：大电流回路 → 环路面积最小化
优先级3（必须）：反馈信号 → 远离电感和开关节点
优先级4（重要）：模拟信号 → 包地保护，远离数字线
优先级5（重要）：时钟/SPI信号 → 串联匹配电阻，等长
优先级6（一般）：GPIO/控制信号
```

#### 5.2.3 信号完整性检查清单
```
□ 电源平面完整，无大面积割裂
□ 去耦电容到IC引脚的过孔数 ≥ 2（降低寄生电感）
□ 模拟信号与数字信号无交叉
□ 模拟地与数字地单点连接（0Ω电阻/磁珠）
□ 差分信号走线等长等距
□ 高速信号（SPI>1MHz）长度 < 5cm
□ 晶振/振荡器周围无其他走线
□ 反馈电阻靠近DC-DC FB引脚
□ 所有悬空输入已处理（上拉/下拉/接地）
□ 丝印标注清晰完整
```

#### 5.2.4 热设计检查
```
□ 功率器件有足够的铜皮散热
□ 热敏感元件（精密电阻/基准）远离热源
□ 过孔阵列辅助散热（如需要）
```

#### 5.2.5 制造可行性检查
```
□ 最小线宽 ≥ 6mil (0.15mm)
□ 最小线距 ≥ 6mil (双层板可做)
□ 过孔内径 ≥ 0.3mm，外径 ≥ 0.6mm
□ 丝印字体高度 ≥ 0.8mm
□ 板卡名称 + 队伍编号 在顶层覆铜层
```
