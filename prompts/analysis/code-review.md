# 代码审查 Prompt

> 用于审查代码质量、发现潜在问题

## 审查维度

1. **功能性**：逻辑是否正确，边界条件是否处理
2. **可读性**：命名是否清晰，结构是否合理，注释是否充分
3. **性能**：是否有不必要的计算/循环/请求
4. **安全**：是否有注入风险/权限漏洞
5. **可维护性**：是否便于后续修改和扩展

## 输出格式

| 问题 | 严重程度 | 位置 | 建议 |
|------|---------|------|------|
| xxx | 高/中/低 | 文件名:行号 | 修改建议 |

- 严重问题（高）：必须修改
- 建议优化（中/低）：可选择修改
