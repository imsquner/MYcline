# 图像识别 Prompt

> 用于在其他 AI 工具（支持多模态的模型）中完成图片识别后，再将结果带回 MYcline 使用。
> 注意：DeepSeek-V4-flash 不支持直接识别图片，此 prompt 供你在 GPT-4V / Claude Vision / Gemini 等工具中使用。

---

## 📌 使用流程

```
你看到一张需要识别的图片
       │
       ▼
① 复制此 prompt → 粘贴到支持识图的 AI（如 Claude/GPT-4V/Gemini）
② 上传图片
③ AI 输出结构化识别结果
④ 把结果复制回 MYcline，让 Cline 继续处理
```

---

## 🧩 标准识图 Prompt（复制以下内容到识图 AI）

```
你是一位专业的图像分析专家。请严格按以下要求识别我上传的图片：

## 一、基本信息提取

请输出以下字段（如不适用则写"无"）：
- 图片类型：截图/照片/图表/文档/UI界面/其他
- 主要语言：中文/英文/其他/无文字
- 画面元素：列出图片中所有可见元素（人物/物体/文字区域/按钮/图标等）

## 二、文字内容识别（OCR）

1. 提取图片中所有文字，保持原始排版
2. 如果是代码截图 → 直接输出完整代码，标注语言
3. 如果是文档/笔记截图 → 全文转录，标注段落结构
4. 如果是 UI 界面 → 提取所有按钮文本、标签、提示文字
5. 如果是图表 → 提取标题、坐标轴标签、数据点值

## 三、结构化输出

### 如果是代码/技术相关：
```language
[完整代码，保留缩进和换行]
```

### 如果是文档/文字：
> [逐段转录内容]

### 如果是 UI/界面：
| 元素类型 | 文本内容 | 位置描述 |
|---------|---------|---------|
| 按钮 | "提交" | 右下角 |
| 输入框 | "请输入姓名" | 页面顶部 |
| ... | ... | ... |

### 如果是数据/图表：
| 维度 | 值 | 备注 |
|------|---|------|
| ... | ... | ... |

## 四、额外说明

- 如果图片模糊或文字不清：标注"【不确定】"在对应位置
- 如果有图表趋势：描述变化方向和数据关系
- 如果有逻辑关系（流程图/架构图）：用 Markdown 或 Mermaid 还原
- 如果有颜色/样式信息：标注色值或样式描述

请开始分析。
```

---

## 🔄 识别完成后回传格式

当你在识图 AI 得到结果后，粘贴到 Cline 时，使用固定格式包裹：

```

还有这个识图结果，你帮我处理一下：

[把识图 AI 的输出粘贴在这里]

```

这样 Cline 就能通过任务分类规则自动识别为 `analysis/image-analysis`，并知道如何接续处理。

---

## 📂 此 prompt 的定位

| 项目 | 内容 |
|------|------|
| 分类 | analysis / image-analysis |
| 使用者 | 你先在其他识图 AI 用 → 结果带回 Cline |
| 原因 | DeepSeek-V4 不支持多模态，需中转 |
| 后续扩展 | 如果你后续接入了支持识图的 MCP 服务器，可以跳过此步骤 |
