# 【写作规则】电路设计文档 —— CIMC硬件设计文档

> 从电源板文档编写经验中提取的通用规则
> 适用于PT100板、精密电阻测试板等硬件设计文档

---

## 核心原则

### 1. 不说空话，多说"为什么"和"如果不这样会怎样"
- ✅ "选2.5kΩ是为了凑1mA恒流源，因为赛题要求1mA±0.5%"
- ❌ "本设计采用2.5kΩ电阻实现恒流源功能"

### 2. 以"模块化思路"组织，不是"信号流"
- 每个模块独立成节：为什么选这个器件 → 参数怎么算 → 如果换一个值会怎样
- 避免用"信号链"、"架构"这种抽象词

### 3. 计算过程要"可验证"
- 公式 → 代入数值 → 计算结果 → 结果分析
- 让评委可以拿计算器复算

### 4. 少说"满足要求"，多说"余量多少"
- ✅ "峰值电流2.29A，电感额定3.5A，余量35%"
- ❌ "满足要求"

### 5. 表格要"可比较"
- 同一参数在不同条件下的对比（不同温度/不同输入电压）
- 不要只列一个值

### 6. 避免"赛题评分对应"章节
- 评分要求融入各模块正文中自然体现
- 不要单独列一个表格硬对应

---

## 文档中每个模块的标准结构

每个功能模块按以下格式写：

```
### 模块名（如"恒流源电路"）

1. **要解决什么问题** → 一句话说明
2. **选了什么方案** → 核心器件+关键参数
3. **参数是怎么算出来的** → 公式→数值代入→结果
4. **如果换一个值会怎样** → 对比分析（可选）
5. **实际选型余量检查** → 额定值vs实际值
```

## 电源板文档已确认的写法示范

电源板文档1的正确写法特征：
- 段段都有"为何这样选"的决策依据
- 宽电压分析做成对比表格
- 计算过程全部展开（公式→数值→结果）
- 没有"信号链架构"、"赛题评分对应"等抽象章节
- 每个模块都能独立阅读
